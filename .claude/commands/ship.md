# /ship — SocraChat 변경사항 배포 플로우

수정 완료 후 아래 순서로 자동 실행한다.

## 1. i18n 키 누락 검사
`storybook.html`에서 `data-i18n` 속성으로 등록된 키를 모두 추출한다.
추출한 키가 `window.__SC_I18N` 딕셔너리의 `ko`, `ja`, `en` 세 곳에 모두 존재하는지 확인한다.
누락된 키가 있으면 사용자에게 알리고 수정 후 계속 진행한다.

## 2. 브라우저 확인 리마인드
자동화 불가. 아래 항목을 체크했는지 사용자에게 확인한다:
- [ ] SC / AP 테마 전환
- [ ] KO / JA / EN 언어 전환
- [ ] Light / Dark 모드
- [ ] Desktop / Mobile 뷰포트

사용자가 확인했다고 하면 다음 단계로 넘어간다.

## 3. git 커밋 & 푸시
변경된 파일을 확인하고 적절한 커밋 메시지를 작성해 커밋 후 푸시한다.
커밋 메시지는 변경 내용을 기반으로 자동 작성한다.

## 4. Vercel 배포
```
vercel --yes --prod
```
배포 완료 후 프로덕션 URL을 알려준다: https://socrachat.vercel.app

## 5. 피그마 비교 여부 판단
아래 중 하나에 해당하면 피그마 비교/재캡처를 제안한다:
- 화면 레이아웃이나 컴포넌트 구조가 크게 바뀐 경우
- 스테이크홀더 공유 예정인 경우
그 외엔 "피그마는 다음 공유 시점에 업데이트" 안내로 마무리한다.
